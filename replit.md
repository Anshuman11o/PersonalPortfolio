# Personal Portfolio Website

## Overview

This is a full-stack personal portfolio website built using a modern tech stack with React frontend, Express.js backend, and PostgreSQL database. The application showcases projects, work experience, and provides contact information for a computer science student specializing in AI, robotics, and software development.

## User Preferences

Preferred communication style: Simple, everyday language.

## System Architecture

### Frontend Architecture
- **Framework**: React 18 with TypeScript
- **Routing**: Wouter for lightweight client-side routing
- **State Management**: TanStack Query for server state management
- **UI Framework**: Radix UI components with shadcn/ui design system
- **Styling**: Tailwind CSS with custom theme configuration
- **Animations**: Framer Motion for smooth transitions and animations
- **Build Tool**: Vite for fast development and optimized builds

### Backend Architecture
- **Runtime**: Node.js with TypeScript
- **Framework**: Express.js for RESTful API endpoints
- **Database ORM**: Drizzle ORM for type-safe database operations
- **Database**: PostgreSQL (configured for Neon serverless)
- **Session Management**: Express sessions with PostgreSQL store
- **Development**: tsx for TypeScript execution in development

### Data Storage
- **Primary Database**: PostgreSQL with Drizzle ORM
- **Schema**: Separate schema file in shared directory for type safety
- **Migrations**: Drizzle-kit for database schema management
- **Fallback**: In-memory storage implementation for development

## Key Components

### Database Schema
- **Projects Table**: Stores portfolio projects with title, description, GitHub URLs, and images
- **Experiences Table**: Stores work experience with company, role, description, and date ranges
- **Users Table**: Basic user management (currently in memory storage)

### Frontend Pages
- **Home/About**: Hero section with personal introduction and call-to-action buttons
- **Projects**: Grid layout showcasing portfolio projects with modal details
- **Experience**: Timeline of work experience and internships
- **404 Page**: Custom not found page with helpful messaging

### UI Components
- **Navigation**: Responsive navbar with mobile sheet menu
- **Cards**: Reusable project and experience cards with hover effects
- **Dialogs**: Modal dialogs for detailed project/experience views
- **Forms**: Form components with validation support (React Hook Form ready)

### External Integrations
- **Email**: SendGrid integration for contact forms
- **Icons**: React Icons for social media and UI icons
- **Responsive Design**: Mobile-first approach with Tailwind CSS

## Data Flow

1. **Client-Side Rendering**: React components fetch data using TanStack Query
2. **API Routes**: Express.js handles RESTful endpoints with `/api` prefix
3. **Database Operations**: Drizzle ORM manages PostgreSQL interactions
4. **State Management**: TanStack Query caches server responses and manages loading states
5. **Error Handling**: Centralized error boundaries and toast notifications

## External Dependencies

### Development Tools
- **TypeScript**: Full type safety across frontend and backend
- **ESBuild**: Fast production builds for server code
- **Vite**: Development server with hot module replacement
- **PostCSS**: CSS processing with Tailwind and Autoprefixer

### UI Libraries
- **Radix UI**: Headless component primitives for accessibility
- **Tailwind CSS**: Utility-first CSS framework
- **Framer Motion**: Animation library for smooth interactions
- **React Hook Form**: Form state management and validation

### Backend Services
- **Neon Database**: Serverless PostgreSQL hosting
- **SendGrid**: Email delivery service
- **Session Storage**: PostgreSQL-backed session management

## Deployment Strategy

### Build Process
1. **Frontend Build**: Vite compiles React app to static assets in `dist/public`
2. **Backend Build**: ESBuild bundles server code to `dist/index.js`
3. **Database**: Drizzle pushes schema changes to PostgreSQL

### Environment Configuration
- **Development**: Uses tsx for TypeScript execution and Vite dev server
- **Production**: Compiled JavaScript served by Express with static file serving
- **Database**: Environment variable `DATABASE_URL` for PostgreSQL connection

### File Structure
- **Client**: Frontend React application in `client/` directory
- **Server**: Backend Express application in `server/` directory  
- **Shared**: Common TypeScript types and schemas in `shared/` directory
- **Migrations**: Database migration files generated by Drizzle

The application follows a monorepo structure with clear separation between client, server, and shared code, enabling type safety across the full stack while maintaining scalability and maintainability.
